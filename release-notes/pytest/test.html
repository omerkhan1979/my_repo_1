This resolves the bug where orders where not re-enqueued when the "Clear Session" functionality was utilized. This tool will now work as expected: after selecting "Clear Session", the order is unassigned from the current picker and then re-enqueued into the manual queue so that another user is able to assign/pick the order.

Improvement(s)

OUTBOUND-3341
Version: RT52-22
Product Area(s): Dispatch
A new configuration around the split mechanism has been created in TSC to improve the creation of new environments. This method no longer requires creating a Pull Request (PR)/deployment. Additionally as part of this increment, orders created with no recognized/in-stock lines will be moved to `served` status immediately after the order splits.

OUTBOUND-3129
This new shipping label display was configured for specifically for SMU (LAM001) based on the clients feedback. The main changes are:
- Removed concatenation `DE+zone-id+zone-seq-num`
- Reduced emphasis on the spoke's ID
- Increased emphasis on the route's ID
- The route's stop number was added under the route's ID
- The date format is DD/MM/YYYY
- The order's ID length is maxed to 16 characters

INC-769
This feature relates to the case, when new user tries to signin to TOM, but had not yet received a role. Instead of a page with error, it sees a nice page explaining that it must be granted a role to use TOM UI.

INBOUND-2281
Starting in this release, as part of the Known Address Infrastructure (KAI) project, we have added additional logic to ensure manual area adjustments only occur on known addresses within the MFC. Please note, this logic is hidden behind a Feature Flag coupled with the KAI project.
